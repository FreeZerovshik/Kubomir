# Кубомир — добавить блок (OCP, без правки движка)

Блоки — data-driven. Новый блок = **3 точки** в `js/world.js` (+ опц. предмет в `js/items.js`). Не дублируй логику.

## Шаги

1. **`BLOCKS` (js/world.js).** Добавь строку в массив (индекс = значение в `obj[]`; **дописывай в конец**, не меняй порядок — сломаешь сейвы):
   ```js
   { id:"<id>", name:"<рус>", solid:<true|false>, hardness:<0..25>, drop:"<itemId|null>", yield:<n>, deco?:true }
   ```
   `solid:true` → коллизия. `hardness>0` → добывается (время = `hardness*0.34/toolPower`). `deco:true` → не-твёрдый декор.

2. **`drawObj` (js/world.js), новый `case "<id>":`.** Рисуй **снизу вверх** (низ-якорь: `by = ty*TILE+TILE-2`, центр `cxp`). Тень для solid уже рисуется до switch. Детали — только детерминированно через `G.ihash(tx,ty,salt)` (иначе мерцание). Образцы рядом: `tree`, `rock`, `coal`/`iron` (валун+крапины), `woodblock`/`stonebrick` (кубы-постройки).

3. **Размещение в `World.gen` (js/world.js).** В нужном биоме (`gnd===GROUND.grass|stone|sand|snow`) выстави объект детерминированно: порог по `ihash(tx,ty,seed+SALT)` (россыпь) или `smooth(tx*f,ty*f,seed+SALT)` (жилы/кластеры). Уникальный SALT.

4. **Если новый дроп — предмет (js/items.js).** Добавь в `ITEMS` (`{id,name,icon,stack, place?:"<blockId>"}`). `place` → блок можно ставить из хотбара. Иконка-эмодзи или процедурная.

5. **Проверка:** скил `cubeworld-playtest` (eval найдёт блок у спавна и добудет; скриншот — если важен вид).

## Gotchas
- Низ-якорь рисования (объекты >1 тайла растут вверх, крона выше `baseY` для y-сортировки).
- Только `G.ihash`, никакого `Math.random()` в `drawObj`/`gen`.
- Дописывай в `BLOCKS`/`ITEMS` в конец (индексы — в сейвах).
- Сброс — не нужен; `World.edit` уже пишет правку в `_edits` для сейва.
