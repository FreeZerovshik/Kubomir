# Кубомир — Project Rules

2D-Minecraft с видом сверху. iPad/web, HTML5 Canvas 2D, vanilla JS. Русский UI. **No build step, no images** — всё процедурно. Стек взят из Mars (`../Mars`). **Аудитория — все возрасты:** доступно новичку и ребёнку (Лев, 6 лет), но с настоящей глубиной для игроков постарше и фанатов Minecraft. Принцип **«easy to learn, hard to master»**: дружелюбный onboarding, крупные кнопки и мирный старт — НО глубину НЕ вырезаем. Её регулируют **сложность** (мирный/лёгкий → обычный/хард) и опциональные слои контента. Игра НЕ «детская» — она для всех.

## Язык общения

- **Всегда отвечай пользователю по-русски** (чат, объяснения, summary).
- Код, имена переменных, тех-термины (commit, eval, tile, chunk) — как есть.

## File Layout (модульный — НЕ «god-файл»)

- `index.html` — cache-busted загрузчик (`?t=Date.now()`). Правки в `js/*.js` применяются по перезагрузке браузера.
- `js/engine.js` (~330 строк) — глобал `G`: состояние, **камера**, ввод+джойстик, день/ночь, звук, частицы, тряска, реестр сцен, сейв, главный цикл.
- `js/world.js` (~280) — генерация острова (value-noise), тайлы, таблица `BLOCKS`, рендер с y-сортировкой, коллизии, `G.drawSteve`.
- `js/scenes.js` (~280) — сцены `menu` + `world` (игрок, добыча, HUD). Новые сцены — сюда.
- `js/main.js` — bootstrap (letterbox 4:3). Не трогать без причины.
- `css/style.css` — полноэкранный canvas, `touch-action:none`. Не трогать.

Файлы небольшие — можно читать целиком, но для точечных правок сперва `grep -n`.

## Coding Conventions

- **No image assets** — `ctx.fillRect`, `ctx.arc`, gradients, paths.
- **IIFE + глобал `G`**: каждый файл `(function(){ "use strict"; const G = window.G || (window.G = {}); ... })()`.
- **Хелперы** (в `G`): `G.f(px, weight?)` шрифт, `G.rr(ctx,x,y,w,h,r)` rounded-rect (только path — потом сам `fill()`/`stroke()`), `G.rectHit(rect,x,y)`, `G.clamp/lerp/dist`, `G.PI`, `G.VIEW` {w:1024,h:768}, `G.TILE` (48), `G.PAL` (палитра).
- **Data-driven таблицы** вместо `if/else`: `BLOCKS`, `GROUND` (и будущие `ITEMS`/`RECIPES`/`MOB_KINDS`).
- **Контракт сцены**: `{ isWorld?, useJoystick?, enter(), update(dt), draw(ctx), onTap(x,y), onPointerDown/Up/Move(x,y), onAction() }`. Регистрация через `G.addScene(name, scene)`, переход `G.go(name, instant?)` (fade).

## Ключевые API движка (что переиспользовать, а не писать заново)

- **Камера**: `G.cam {x,y}`, `G.worldToScreen(wx,wy)`, `G.screenToWorld(sx,sy)`. Мир рисуется внутри `ctx.translate(-cam.x,-cam.y)`; HUD/кнопки/джойстик — в экранных координатах ПОСЛЕ `restore()`.
- **Мир** `G.World`: `gen(seed)→spawn{x,y}`, `gTile(tx,ty)`, `oTile(tx,ty)`, `edit(tx,ty,v)` (правит + пишет в `_edits` для сейва), `solidPx(wx,wy)`, `drawVisible(ctx, sprites)`, `BLOCKS`, `OBJ` (id→индекс), `GROUND`, `W`/`H`.
- **Шум**: `G.ihash(x,y,salt)` → 0..1 детерминированно (стабильная «текстура»). Внутри — `smooth`/`fbm`.
- **Персонаж**: `G.drawSteve(ctx, x, y, face, walk, swing)` — блочный Стив, вид сверху ¾.
- **День/ночь**: `G.dayCycle` (0..1), `G.sunHeight()` (-1..1), `G.daylight()` (0..1), `G.dayNightOverlay(ctx)`. Цикл `DAY_PERIOD≈300с`, идёт только в `isWorld`-сценах.
- **FX/juice**: `G.fx.burst(x,y,color,n,spd,life)`, `G.fx.ring(...)` (МИРОВЫЕ координаты — рисуй внутри translate), `G.shake(amt)` (5–14), `G.hitStop(sec)` (0.03–0.05).
- **Звук**: `G.audio.dig/pop/pickup/blip/hit/tone(...)`. Только через `G.audio.*`, не дёргай AudioContext напрямую.
- **Ввод**: `G.moveDir()→{x,y,mag}` (клавиши+джойстик), `G.input.keys`, динамический джойстик (`useJoystick:true` + `G.drawJoystick`), `onTap` = короткое касание. `G.uiBlock.push(rect)` каждый кадр в `update` — чтобы джойстик НЕ стартовал на кнопках.
- **Сейв**: `G.saveGame/loadGame/hasSave/clearSave`, ключ `cubeworld_save_v1`. Формат `{state, dayCycle, edits}` — мир регенерится из `state.seed`, потом применяются `edits`. `loadGame` мёржит в `defaultState()` (старые сейвы не ломаются).

## Модель мира

- **Два неявных слоя на тайл**: `ground` (вода/песок/трава/земля/камень/снег — `GROUND`) и `obj` (объект, 0=пусто — индекс в `BLOCKS`). Хранятся в `Uint8Array`.
- `BLOCKS[i] = {id, name, solid, hardness, drop, yield, tall?, deco?}`. `solid`+вода → коллизия (`solidPx`). `hardness>0` → добывается.
- **Сохраняем только правки** (`World._edits["tx,ty"]→objVal`), не всю сетку.

## Добыча

- Время ломания = `hardness * BASE_BREAK / toolPower` (`BASE_BREAK=0.34`, `toolPower=3` пока фикс — кирка по умолчанию; тиры дерево/камень/железо/алмаз — позже).
- Удержание «ДОБЫТЬ»/`E`/`Space` → `nearestMinable()` (ближайший solid-объект, приоритет по взгляду `fx/fy`). Тап по блоку → разовая цель. `breakBlock` → `World.edit(...,0)` + дроп в `state.resources` + `fx.burst` + `shake`+`hitStop`.
- **Сброс таймеров в 0** при завершении (`mineT`, `swingT`).

## Архитектура, SOLID, паттерны (прагматично, под canvas-игру)

Как в Mars: SRP (рендер ≠ логика ≠ ввод), **OCP через данные** (новый блок/предмет/моб = строка таблицы + draw-case, без правки движка), DIP (зависим от `G.fx/audio/go/World`, не от DOM/ctx из логики). Паттерны: **Factory** (будущие `makeMob`), **Registry** (`G.addScene/go`), **Strategy via data** (таблицы), **Entity list** (массивы с обратным циклом `for(i=len-1;i>=0;i--)` для удаления).

Анти-паттерны: magic numbers без имени в новой логике (разовые координаты рисования — ок инлайном); god-функции >~80 строк; дублирование 3+ раз (выноси в хелпер); скрытое состояние помимо `G.state`/`s.data`.

При рефакторинге: сперва корректность, потом красота. Крупные структурные изменения — согласовывать.

## Gotchas (не наступать)

1. **Два пространства координат.** Мир рисуется в `ctx.translate(-cam.x,-cam.y)` (мировые px). HUD/кнопки/джойстик — экранные, ПОСЛЕ `restore()`. Тап (экранный) → мир через `G.screenToWorld`. `G.fx` — мировые.
2. **y-сортировка.** Объекты и игрок рисуются по `baseY` (низ). Спрайт игрока: `baseY = py+16`. Объекты рисуются от низа тайла вверх (`by = ty*TILE + TILE - 2`), крона — выше `baseY` (игрок корректно за/перед деревом).
3. **Только детерминированная «текстура».** Никакого `Math.random()` в рендере/пер-тайл генерации (мерцание!). Используй `G.ihash(tx,ty,salt)`. `Math.random` ок для разовых вещей (выбор сида, направление частиц).
4. **Не оставляй таймеры отрицательными** — сбрасывай в 0 (как в Mars: `if(!s.x)` ломается на `-0.01`).
5. **Коллизия** = `World.solidPx` (solid-объект ИЛИ вода). Движение по осям раздельно (`blocked(nx,py)` потом `blocked(px,ny)`), игрок зажат в `[16, W*TILE-16]`.
6. **`uiBlock` каждый кадр** в `update` (`G.uiBlock.push(actRect, menuRect)`) — иначе джойстик стартует на кнопке.
7. **Сейв = сид + правки**, не вся сетка. `World.edit` уже пишет в `_edits`.
8. **Добавление блока (OCP):** строка в `BLOCKS` → `case` в `drawObj` (низ-якорь!) → размещение в `World.gen`. Не дублируй логику.
9. **Дроп ресурса** через `state.resources[drop] += yield`. День растёт по переходу `dayCycle` через 0 (полночь) в `world.update`.

## Token-Saving Rules

- **Читай с умом.** Файлы небольшие, но для точечной правки — `grep -n "pattern" js/scenes.js` сперва. Не `cat`/`sed` — используй `Read` с `offset`/`limit`.
- **Проверяй через eval, не скриншоты.** `mcp__Claude_Preview__preview_eval` с JSON-отчётом (≈дёшево) > `preview_screenshot` (~1500 ткн). Скриншот — только для визуала (новые иконки, анимация, раскладка). См. скил `cubeworld-playtest`.
- **Симулируй действия в eval**: `G.World.gen(seed); G.go("world",true); for(let i=0;i<240;i++) s.update(0.016)` — дешевле драйва браузера. Очищай врагов `s.mobs.length=0` для пост-боевых стейтов.
- **Edit (string-replace) для правок ≤~50 строк**, Write — для новых файлов/полных переписей. Не пере-`Read` только что отредактированный файл.
- **Игровой цикл засыпает при скрытой вкладке** (rAF throttling) — если стейт не меняется в eval, гоняй `s.update(dt)` вручную в цикле.

## Баланс (текущий)

- `TILE=48`, мир `200×200`, полный день `~5 мин` (день ~60%/ночь ~40%).
- Твёрдость: дерево 3, камень/уголь 5, железо 6, кактус 2. Дроп: дерево→3 🪵, остальное→1.
- Планируемые тиры инструментов (power): рука 1, дерево 2, камень 4, железо 6, алмаз 9.
- Уровни сложности (как в Mars — `G.DIFFICULTY`, скоро): **мирный/лёгкий** (младшие — слабее/нет мобов, щадящий голод) → **обычный/хард** (ветераны — настоящее выживание). Так одна игра служит и Льву, и взрослым. Дефолт — доступный, но не «детский».

## Roadmap (фазы)

- **Готово:** тех-каркас — мир+камера, генерация, рендер, добыча, HUD, день/ночь, сейв.
- **Фаза 2:** инвентарь + хотбар (слоты, выбор предмета, `ITEMS`).
- **Фаза 3:** крафт (`RECIPES`) + верстак + печь; строительство (ставить/ломать блоки).
- **Фаза 4:** HP + голод; мобы ночью (`MOB_KINDS`: зомби/паук/скелет/крипер — форк ИИ Mars); бой (мах кирки/меча, dash).
- **Фаза 5:** пещеры (вход-тайл → отдельная зона), руды/свет/факелы, сундуки, кровать.
- **Фаза 6:** алмаз, больше блоков/мобов, мирные звери+еда, биомы (снег/пустыня).
- **Фаза 7:** музыка по биомам, журнал заданий, звание-финал, PWA для iPad (manifest/sw/icons), полировка.
